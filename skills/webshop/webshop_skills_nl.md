### GENERAL SKILLS ###

Here are some useful general strategies for completing tasks in this environment:

1. Prioritize Core Keywords: Include the product type, 1-2 key functional attributes, and any hard constraints (price, size, color) in the search query. Omit secondary descriptors to avoid over-constraining. Apply this before issuing the first search or when refining an over-specific query that yields few or irrelevant results.

2. Iterative Query Refinement: If top results are irrelevant, adjust keywords (add or remove terms or use synonyms) or apply site filters instead of repeating the same query. Apply this after a search results page shows mostly off-target items or repeats despite multiple identical searches.

3. Scan Before You Click: Read product titles, thumbnails, and prices in the results list to ensure the item plausibly meets core constraints before opening it. Apply this on any search results page when deciding which product link to open next.

4. Verify Early, Abort Fast: On the product page, immediately check category, core attributes, and price. If a key constraint is violated, leave the page at once. Apply this within the first observation on every product detail page.

5. Set Mandatory Variants: Always select required variant options (size, color, capacity, etc.) before evaluating price or purchasing. Apply this after confirming the product type matches and before any purchase action.

6. Confirm Hidden Attributes: Open the Description or Features section to ensure non-visible specs (e.g., material, wash method, slip resistance) meet constraints. Skip item if unconfirmed. Apply this when a constraint is not evident from the title, images, or variant list.

7. Check Variant Pricing: For products showing a price range, select the exact variant combination and verify its specific price is within budget before buying. Apply this whenever the price is shown as a range or changes with variant selection.

8. Avoid Option Loops: If changing variants yields no path to meeting all constraints, stop toggling and return to search rather than looping on the same product. Apply this after two to three unsuccessful variant changes on the same page.

9. Use Result Pagination: If the first result page lacks suitable items, navigate to additional pages or adjust sorting and filters before concluding unavailability. Apply this when initial results are insufficient and additional pages are indicated.

10. Respect Constraint Hierarchy: Treat product type, category, and hard limits (price, size, color) as non-negotiable. Secondary preferences can be used to rank but not override these. Apply this whenever evaluating whether to open or purchase a candidate item.

11. Conclude When Necessary: After exhaustive search and reasonable refinements yield no matching product, stop and report failure instead of endless retries. Apply this when multiple varied searches and pages produce no viable candidates within remaining steps.

12. One Good Click Rule: Limit inspection of clearly mismatched items to a single quick check. Prioritize exploring new candidates over re-opening the same unsuitable product. Apply this if a product has already been deemed unsuitable once.

13. Purchase Decisively: Once all constraints are confirmed on a variant and price fits, execute Buy Now without unnecessary further navigation. Apply this after validating every constraint on the current product variant.

14. Documented Attribute Search: When a critical attribute (e.g., tuxedo, butt lifting) is not filterable, keep the query concise but include that keyword to improve precision. Apply this during initial or refined searches targeting attributes not covered by site filters.

15. Fallback to Best Match: If no product meets every constraint, consider nearest matches that satisfy all hard constraints and most secondary ones, noting any unmet soft attributes. Apply this near task limit when exact match is absent but partial matches exist.

### TASK: apparel ###

For this apparel shopping task, apply these specific strategies:

1. Focus Key Query: Include only product type plus must-have attributes (gender, garment, fabric, sleeve or fit, price cap) in the search. Drop minor or redundant terms to widen relevant results. Apply this before issuing or refining any search query.

2. Validate Core Match: On every product page, confirm the title and category matches the requested garment type and primary attributes. If not, immediately return to results. Apply this right after opening a product detail page.

3. Set Variants First: Always select the required size and color options before evaluating price or purchasing to ensure the exact variant meets constraints. Apply this upon landing on a relevant product page with selectable options.

4. Check Variant Price: After choosing size and color, verify the displayed price for that specific variant is within budget. Abandon if it exceeds the limit. Apply this immediately after variant selection and before clicking buy.

5. Confirm Hidden Specs: Scroll to bullet points or description to verify non-title attributes (material, care, fit, design features). Reject items lacking evidence. Apply this before purchase on a candidate product where attributes are not obvious in the title.

6. Adaptive Search Loop: If top results are irrelevant, alter search by removing rare constraints or scanning more pages instead of repeating identical queries or cycling one bad product. Apply this after an initial search yields no suitable items or only mismatches.

### TASK: footwear ###

For this footwear shopping task, apply these specific strategies:

1. Load Search Constraints: Include all hard filters (type, function, color, size, price cap) directly in the initial search query to surface only viable footwear candidates. Apply this before the first search or whenever starting a new search cycle.

2. Verify Key Features: Open the product description or specs to explicitly confirm required functional attributes (e.g., slip-resistant, rubber sole) before moving toward purchase. Apply this immediately after opening a product page and before selecting variants or buying.

3. Select Variants First: Always choose the requested color and size options on the product page prior to any purchase action to ensure the exact variant meets constraints. Apply this once a candidate product page is open and verified for required features.

4. Check Variant Price: Confirm the displayed price for the chosen size and color is strictly below the user's maximum before clicking Buy Now, especially when ranges are shown. Apply this after selecting variants and just before initiating checkout.

5. Exit Non-Matches Fast: If a product or result page clearly violates any hard constraint, back out immediately instead of toggling options or repeating identical searches. Apply this upon noticing missing attributes, wrong category, unavailable size or color, or over-budget price.

### TASK: home_decor ###

For this home decor shopping task, apply these specific strategies:

1. Encode All Constraints: Include product type plus every known color, size, shape, feature, and price limit in the initial search query to surface the most relevant home-decor items. Apply this whenever beginning a search or when current results lack items meeting multiple stated attributes.

2. Verify Product Category: Before investing time configuring options or purchasing, ensure the item's category (e.g., pillow, rug, furniture set) matches the user's requested product type and room usage. Apply this immediately after opening a product page from search results.

3. Use Variant Selectors: Systematically choose color, size, shape, and other visible variants on the product page, confirming that each selection updates the listing to match user constraints and price. Apply this after landing on a potentially suitable product that offers configurable attributes.

4. Open Details For Hidden Specs: Expand Description, Features, or Specs tabs to confirm less-visible requirements (e.g., material, washability, printing method) that are not obvious from titles and thumbnails. Apply this when any user constraint is not directly visible in the main listing or variant options.

5. Avoid Product Loops: If a candidate fails any constraint after inspection, return to results or refine the query. Do not repeatedly reopen or re-filter the same non-qualifying product. Apply this after verifying a product and finding a mismatch with any required attribute.

6. Confirm All Then Commit: Proceed to Buy Now only after every user constraint (type, attributes, and price) has been positively confirmed. Otherwise continue searching. Apply this right before clicking the purchase button on a configured product page.

### TASK: electronics ###

For this electronics shopping task, apply these specific strategies:

1. Constraint-Rich Search: Pack product type plus every mandatory attribute (features, color, size, price cap) into the initial search string to surface only highly relevant electronics items. Apply this whenever starting a new product hunt or refining after poor results.

2. Configure In-Page Variants: Once on a promising product page, immediately set color, size or pack, and other variant selectors to test if the SKU can meet all requested specs. Apply this right after opening a candidate item from search results.

3. Hard-Gate Price and Specs: Before proceeding to buy, cross-check the displayed title, specs, and price. Abandon any item that exceeds the budget or lacks the exact requested feature set. Apply this after configuring variants but before clicking Buy Now.

4. Bail on Mismatch Fast: If variant clicks do not update the product details to the required spec, or if the page remains wrong, use Back to results and seek another item instead of retrying the same option. Apply this when repeated option clicks leave title or specs unchanged or incompatible.

5. Prioritize High-Match Results: Scan the result list for items whose titles show all key constraints. Open those first, rather than settling for the earliest but partial match. Apply this on any search results page with multiple potential electronics products.

6. Avoid Click Loops: Track visited products and queries. If progress stalls, refine the search or change items instead of cycling between the same pages. Apply this after revisiting a page or running an identical search without new insights.

### TASK: accessories ###

For this accessories shopping task, apply these specific strategies:

1. Constraint-Packed Search: Compose the initial search query by combining the accessory type with every known constraint (color, size, price cap, material, washability) to surface only viable candidates. Apply this before the first search when the user provides multiple specific requirements.

2. Explicit Variant Selection: Always click and visibly confirm the exact color, size, or pattern variant on the product page instead of assuming the default matches the request. Apply this immediately after opening a product that offers multiple variant options.

3. Function Fit Check: Read the title and description to verify the accessory's intended use (e.g., eye-shadow case vs. generic makeup bag) aligns with the user's need before proceeding. Apply this right after landing on a product page that seems category-relevant but may have ambiguous use.

4. Post-Config Price Audit: After selecting all variants, re-confirm the displayed price is within the user's budget. If it exceeds, backtrack and search for alternatives. Apply this just before initiating checkout or adding to cart.

5. Fallback to Next Result: If any required variant is unavailable or constraints cannot be met on the current page, return to results and open the next candidate instead of forcing a mismatch. Apply this when the product lacks the needed option or violates any key constraint after inspection.

### TASK: beauty_health ###

For this beauty and health shopping task, apply these specific strategies:

1. Bundle Constraints Search: Combine product type with every user-stated filter (price cap, color, size, material, feature keywords) in one search query to surface highly relevant results. Apply this when the user lists two or more specific requirements (e.g., price plus color, size, feature).

2. Feature-Led Click: Open the first result whose title explicitly mentions the key functional attribute (e.g., double sided, long handle, dry hair) to maximize match likelihood. Apply this after search results appear and at least one headline contains the core feature term.

3. Variant Verification and Set: Within the product page, actively choose color, size, or pack options to ensure the selected variant meets all user constraints before purchase. Apply this when the product offers selectable options and the user has specified any of those attributes.

4. Price Gatekeeping: Cross-check the displayed price (including chosen variant) against the user's maximum. If exceeded, abandon and return to results immediately. Apply this right after selecting the desired variant and before initiating checkout.

5. Minimal Path Purchase: Once a product fully satisfies all constraints, proceed directly to Buy Now without extra browsing to reduce error risk and time. Apply this when all user requirements are confirmed on the current product page.

### TASK: other ###

For this general product shopping task, apply these specific strategies:

1. Embed All Constraints: Build the initial search query to include every required attribute (product type, key features, color or size, and price cap) to surface only high-relevance results. Apply this right before issuing any product search.

2. Confirm Exact Category: Open a result only if its title clearly matches the requested product type, and abandon pages that reveal a different category despite matching other attributes. Apply this immediately upon landing on a product detail page.

3. Select and Verify Variants: After choosing required color, size, or pack options, ensure the page's title and price update to the selected variant before proceeding to purchase. Apply this when a product offers dropdown or swatch selections.

4. Check Fixed Specs: Manually read the description or features to confirm non-selectable requirements (materials, special functions, care instructions) are explicitly met. Apply this before clicking Buy Now on any configured product.

5. Refine If No Match: If initial results do not meet all constraints, adjust keywords or add filters (e.g., synonyms, price slider) instead of re-clicking unsuitable items. Apply this after reviewing a results page that lacks clear matches.

6. Follow Purchase Flow: Maintain the disciplined sequence: Search, Open candidate, Apply constraints, Verify all criteria and price, Buy Now. Avoid unnecessary back-and-forth. Apply this for every shopping task from start to checkout.
